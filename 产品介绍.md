## 一、产品基础定位

| 项目 | 内容 |
|---|---|
| 产品名称 | 琵琶简谱自动生成器（暂定） |
| 产品定位 | 面向琵琶学习者的 AI 辅助练习工具 |
| 目标用户 | 琵琶学习者（核心）、民乐爱好者、音乐教育者 |
| 核心痛点 | 传统琵琶谱获取渠道有限、想弹的流行曲没有适配谱、无法根据自身水平找到合适难度的练习曲 |

| 部署地址 | 待部署（Vercel） |
| 当前版本 | v0.1 — 方案设计阶段 |

## 二、完整功能清单

### 2.1 显性功能

| 功能 | 说明 | 状态 |
|---|---|---|
| 音频/视频上传 | 支持拖拽上传，格式：mp4/mov/mkv / mp3/wav/flac/m4a/aac | 设计中 |
| 音频自动提取 | 视频自动提取音频流，纯音频跳过此步骤 | 设计中 |

| 三难度编曲 | 低/中/高三种难度，涵盖音符密度、节奏、技法、把位、调性、速度六个维度 | 设计中 |
| 古风谱面渲染 | SVG 音符层 + Canvas 宣纸纹理背景，仿古卷轴视觉效果 | 设计中 |
| 琵琶音色播放 | 基于 FluidR3 GM SoundFont 合成琵琶音色，支持播放/暂停/调速 | 设计中 |
| 难度切换 | 生成后一键切换低/中/高三种难度谱面 | 设计中 |

### 2.2 隐性逻辑能力

| 能力 | 说明 |
|---|---|
| 全链路离线运行 | 零外部 API 依赖，音频处理/编曲/渲染均在浏览器端完成 |
| 规则引擎确定性输出 | 同一音频每次生成结果完全相同，不依赖 LLM 随机性 |
| 把位自动优化 | 基于动态规划算法，自动选择最优弦序和把位路径 |
| 技法自动匹配 | 基于音符特征（时值、音程、位置）自动匹配合适技法 |

## 三、AI 技术架构

### 3.1 整体架构

```
用户上传 → ffmpeg.wasm(提取音频) → Basic Pitch(旋律转写) 
         → 规则引擎(三难度编曲) → SVG+Canvas(谱面渲染) 
         → SoundFont(琵琶播放)
```

全链路浏览器端运行，零服务端依赖，零 LLM 调用。

### 3.2 核心技术栈

| 层 | 技术 | 用途 |
|---|---|---|
| 前端框架 | Next.js (React) | 页面路由、组件管理、Vercel 部署 |
| 样式 | Tailwind CSS | 古风 UI 样式 |
| 音频提取 | ffmpeg.wasm | 浏览器端视频转音频、格式转换 |
| 旋律转写 | Basic Pitch (tfjs) | 音频 → 音高序列 + 时值 |
| 编曲引擎 | 纯规则引擎 (TypeScript) | 六维度难度控制 |
| 谱面渲染 | SVG + Canvas | 音符交互 + 古风纹理 |
| 音频播放 | Web Audio API + SoundFont | 琵琶音色合成与播放 |
| 存储 | LocalStorage + JSON | 曲谱库持久化 |

## 四、大模型选型说明

**本产品不使用大模型（LLM）。**

六维度难度编曲参数均可量化为确定性规则（音符密度阈值、技法频率、把位范围等），纯规则引擎即可完成，不引入 LLM 的不确定性和成本。

唯一的专用 AI 模型是 Spotify Basic Pitch（音频转音符），这是专用音频推理模型而非语言模型。

## 七、完整工作流

```
1. 用户进入首页
   ├── 看到水墨动态背景 + 上传入口 + 预置素材快捷选择
   └── 选择操作：
       ├── A) 上传自己的音乐/视频文件
       └── B) 选择一个预置素材 → 跳到步骤2

2. 音频处理（显示进度动画）
   ├── 视频文件 → ffmpeg.wasm 提取音频 → WAV 16kHz mono
   └── 纯音频文件 → 直接进入下一步

3. 旋律转写
   └── Basic Pitch 模型推理 → 输出音符序列

4. 编曲生成（<100ms）
   └── 规则引擎并行生成三种难度简谱

5. 结果展示
   ├── 默认显示「中难度」谱面
   ├── 顶部三标签切换低/中/高
   ├── 底部播放控制栏
   └── 侧边导出按钮

6. 播放体验
   ├── 点击播放 → SoundFont 合成琵琶音色
   ├── 谱面随播放高亮当前音符
   └── 可调速、暂停

7. 保存与导出
   ├── 自动保存到曲谱库（LocalStorage）
   └── 可选导出 PNG / JSON / 打印
```
