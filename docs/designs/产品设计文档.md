# 产品设计文档 — 琵琶简谱自动生成器

> 本文档记录产品详细技术设计，包括架构决策、算法细节、数据模型、迭代计划。面向开发参考和技术复盘。

## 一、产品基础信息

| 项目 | 内容 |
|---|---|
| 产品名称 | 琵琶简谱自动生成器（暂定，待命名） |

| 目标用户 | 琵琶学习者（核心）、民乐爱好者 |

## 三、核心技术决策

### 3.1 技术选型总览

| 决策点 | 候选方案 | 最终选择 | 核心理由 |
|---|---|---|---|
| 前端框架 | Next.js / Vue 3 / 纯 HTML | **Next.js** | Vercel 原生部署、React 生态、SSG 适合展示型页面 |
| 音频提取 | 服务端 ffmpeg / Cloudinary / ffmpeg.wasm | **ffmpeg.wasm** | 浏览器端处理，零服务端成本，离线可用 |
| 旋律转写 | Basic Pitch / Demucs+pYIN / 云 API | **Basic Pitch** | 浏览器端推理，免费开源，拨弦乐器效果好 |
| 编曲引擎 | 纯 LLM / 规则+LLM / 纯规则引擎 | **纯规则引擎** | 六维度可量化，确定性输出，零 API 成本，离线可用 |
| 谱面渲染 | Canvas / SVG / HTML+CSS / SVG+Canvas | **SVG + Canvas 双层** | Canvas 负责古风纹理，SVG 负责交互符号，各取所长 |
| 琵琶音色 | 真实采样 / Web Audio 合成 / SoundFont | **FluidR3 GM SoundFont** | MIT 许可可商用，GM #105 琵琶音色，sf2 浏览器端加载 |
| 曲谱存储 | 服务端数据库 / Supabase / LocalStorage | **LocalStorage + JSON 导出** | MVP 零成本，JSON 导出备份，后续可升级云端 |
| 部署平台 | Vercel / Netlify / Cloudflare Pages | **Vercel** | Next.js 原生支持，免费额度充足 |

### 3.2 关键放弃方案及原因

| 放弃方案 | 放弃原因 |
|---|---|
| LLM 编曲 | 六维度难度控制可量化为确定性规则，LLM 引入不稳定性和成本 |
| Demucs 音源分离 | 用户场景不需要人声/伴奏分离，Basic Pitch 直接输出多音高更高效 |
| 服务端处理 | MVP 零成本起步优先，浏览器端全链路离线可用是差异化亮点 |
| Canvas 独占渲染 | Canvas 像素级渲染无法做单个音符的交互（点击试听、高亮跟弹） |
| 真实琵琶采样 | 免费可商用高质量采样找不到，需后续版本自行录制或合作 |

### 3.3 为什么选择零 LLM 依赖

全链路分析结论：每一步都是确定性计算任务。

```
音频提取 → 信号处理（ffmpeg）
旋律转写 → 模型推理（Basic Pitch，专用音频模型，非 LLM）
编曲改编 → 规则推导（六维度参数化，确定性输出）
谱面渲染 → 图形绘制（SVG + Canvas）
音色播放 → 音频合成（SoundFont + Web Audio API）
```

LLM 唯一可能增值的环节是「编曲改编」中需要音乐理解力的技法选择，但经分析发现技法分配本质是模式匹配（长音→轮指、快重复→扫拂、短促→弹挑交替），规则表可覆盖。架构预留编曲插件接口，未来若规则引擎覆盖不足可插 LLM 插件。

## 四、算法与规则说明

### 4.1 难度降维规则引擎

#### 音符密度降维

```
输入：Note[] (原始旋律音符序列)
输出：Note[] (降维后序列)

低难度算法：
  1. 识别节拍强位置（4/4拍：第1、3拍 > 第2、4拍）
  2. 保留强拍上的音符，删除弱拍装饰音
  3. 连续相同音高合并为单音
  4. 目标密度：每小节 2-4 个音符

中难度算法：
  1. 保留强拍 + 次强拍音符
  2. 删除时值 < 1/16 的快速经过音
  3. 目标密度：每小节 4-8 个音符

高难度：
  完整保留原始音符序列
```

#### 节奏简化

```
低难度：
  - 所有时值量化到八分音符或更长
  - 移除附点节奏、切分音
  - 三连音展开为等时值音符

中难度：
  - 保留附点节奏
  - 保留前八后十六等常见节奏型
  - 移除复杂切分

高难度：
  完整保留原始节奏
```

#### 速度调整

```
低难度：原曲速度 × 0.7 ~ 0.8（60-80 BPM）
中难度：原曲速度 × 0.85 ~ 0.95（80-100 BPM）
高难度：原曲速度（不做调整）
```

### 4.2 技法分配规则

#### 技法-场景映射表

| 触发条件 | 技法 | 符号 |
|---|---|---|
| 时值 ≥ 二分音符 | 轮指 | 丶丶丶 |
| 同音连续快速重复（≥3次） | 扫拂 | 扫/拂 |
| 短促音 + 下一音高跳进 | 弹挑交替 | 丿丶 |
| 下行小二度/大二度 | 绰（下滑） | 绰 |
| 上行小二度/大二度 | 注（上滑） | 注 |
| 持续长音（≥全音符） | 吟揉 | 吟 |
| 音高跳进 ≥ 五度 | 双弹/琵音 | 双/琵 |
| 句尾长音 | 带起/打音 | 带/打 |
| 乐句起始/高潮 | 泛音 | 〇 |

#### 技法频率控制

```
低难度：
  技法池：[弹, 挑, 轮指, 双弹]
  频率：每 4-8 小节插入 1 个技法标记
  算法：遍历小节，每隔 random(4,8) 小节在技法池中随机选一个应用

中难度：
  技法池：[弹, 挑, 轮指, 双弹, 吟揉, 推拉, 琶音, 绰注]
  频率：每 1-2 小节插入 1 个技法标记
  算法：遍历小节，每小节有 50-100% 概率插入技法

高难度：
  技法池：全部 12+ 种
  频率：按原曲需要，每拍都可能出现
  算法：逐音符匹配技法-场景映射表
```

### 4.3 把位优化算法（动态规划）

```
问题定义：
  给定音符序列 N[0..n]，每个音符在不同弦/把位有多个指位候选
  目标：选择指位序列使左手纵向移动距离之和最小

算法（简化动态规划）：
  let dp[i][pos] = 前 i 个音符，第 i 个音符选位置 pos 时的最小移动距离
  
  for i = 1..n:
    for each pos_j of N[i]:
      dp[i][pos_j] = min(
        dp[i-1][pos_k] + distance(pos_k, pos_j)
        for each pos_k of N[i-1]
      )

把位限制（低难度）：
  所有音符强制映射到 D 调一把位（相把 0-6 品）
  可用音高范围：D调自然音阶（一弦一品到四弦六品）

把位限制（中难度）：
  允许三把位以内，即品位范围 0-15
  禁止超过 5 品的快速跳跃

把位限制（高难度）：
  无限制，全品位可用
```

### 4.4 调性适配规则

```
默认输出调性：D 调（琵琶最常用调）

若原始音频非 D 调：
  1. Basic Pitch 输出绝对音高（MIDI note number）
  2. 计算原始调性与 D 调的音程差
  3. 整曲平移音符到 D 调音域
  4. 确保平移后音符都在琵琶音域内（D3 - D7）

琵琶 D 调音域映射：
  空弦：一弦 D4, 二弦 A3, 三弦 E3, 四弦 D3
  一把位音阶：D3 E3 F#3 G3 A3 B3 C#4 D4 E4 F#4 G4 A4 B4 C#5 D5
```

## 五、数据结构设计

### 5.1 核心数据模型

```typescript
// 单个音符
interface Note {
  pitch: number;        // MIDI note number (0-127)
  startTime: number;    // 开始时间（秒）
  endTime: number;      // 结束时间（秒）
  velocity: number;     // 力度 (0-1)，Basic Pitch 输出
}

// 音符（含琵琶指位信息）
interface PipaNote {
  pitch: number;              // MIDI 音高
  stringNumber: 1 | 2 | 3 | 4; // 弦序
  fretPosition: number;       // 品位（0=空弦）
  duration: number;           // 时值（拍数，如 1=四分音符, 0.5=八分音符）
  technique: Technique | null; // 技法标记
  isStrongBeat: boolean;      // 是否强拍
}

// 技法定义
interface Technique {
  type: TechniqueType;  // 技法类型枚举
  symbol: string;       // 简谱符号
  category: 'right' | 'left'; // 右手/左手
}

type TechniqueType =
  | 'tan'      // 弹
  | 'tiao'     // 挑
  | 'lunzhi'   // 轮指
  | 'shuangtan'// 双弹
  | 'sao'      // 扫
  | 'fu'       // 拂
  | 'yaozhi'   // 摇指
  | 'yinrou'   // 吟揉
  | 'tuila'    // 推拉
  | 'chuo'     // 绰（下滑）
  | 'zhu'      // 注（上滑）
  | 'fanyin'   // 泛音
  | 'dai'      // 带起
  | 'da'       // 打音
  | 'pipa'     // 琵音
  | 'guntong'; // 滚/同

// 小节
interface Measure {
  index: number;
  timeSignature: [number, number]; // [4, 4] 表示 4/4 拍
  notes: PipaNote[];
}

// 完整简谱数据
interface ScoreData {
  id: string;                    // 唯一标识
  title: string;                 // 曲名
  sourceType: 'upload' | 'preset'; // 来源类型
  originalAudioHash: string;     // 原始音频哈希
  key: string;                   // 调性（如 "D"）
  tempo: number;                 // 速度 BPM
  difficulty: 'easy' | 'medium' | 'hard';
  measures: Measure[];
  createdAt: string;             // ISO 日期
}

// 曲谱库条目
interface ScoreLibraryEntry {
  id: string;
  title: string;
  thumbnail?: string;            // 可选缩略图（谱面第一行）
  difficulty: 'easy' | 'medium' | 'hard';
  createdAt: string;
  sourceType: 'upload' | 'preset';
  scoreData: ScoreData;
}

// 预置素材条目
interface PresetMaterial {
  id: string;
  title: string;
  category: 'transcribe' | 'reference'; // 转谱素材 / 琵琶经典参考
  audioUrl?: string;             // 素材类：MIDI 音频 URL
  scoreData?: ScoreData;         // 参考类：直接预置的谱面数据
  description: string;
  tags: string[];
}
```

                                ~2-5秒/30秒      <100ms          <500ms加载

### 7.2 SoundFont 播放器设计

```
加载流程：
  1. fetch('FluidR3_GM.sf2') → ArrayBuffer (~140MB，需精简)
  2. sf2Parser.parse(arrayBuffer) → InstrumentPreset[]
  3. 索引 preset #105 (Chinese Pipa)
  4. 构建 sampleMap: { midiNote → AudioBuffer }

精简策略（减小加载体积）：
  - 只提取琵琶音色 (#105) 的音域样本（通常 D3-D7，约 50 个音）
  - 其余 GM 乐器音色丢弃，不加载
  - 精简后预计 < 5MB

播放流程：
  1. 遍历 ScoreData.measures[].notes[]
  2. 每个 PipaNote 计算播放时机：startTime = accumulatedDuration × (60 / tempo)
  3. Web Audio API schedule: sourceNode.start(startTime)
  4. 技法对应的音色变化（轮指→快速重复触发、弹挑→交替触发）

播放与谱面联动：
  - requestAnimationFrame 循环检查当前播放位置
  - 当前播放音符对应的 SVG 元素高亮（fill 颜色变化）
  - 自动滚动到当前小节
```

## 八、边界与异常处理

### 8.1 输入校验

| 校验项 | 规则 | 错误提示 |
|---|---|---|
| 文件类型 | 仅接受视频(mp4/mov/mkv)和音频(mp3/wav/flac/m4a/aac) | 「请上传支持的音频或视频文件」 |
| 文件大小 | ≤ 100MB（浏览器内存限制） | 「文件过大，请上传 100MB 以内的文件」 |
| 音频时长 | 15 秒 ~ 10 分钟 | 「请上传 15 秒到 10 分钟的音频」 |
| 音频内容 | 检测是否包含可识别旋律（Basic Pitch 输出空结果） | 「未检测到可用旋律，请尝试其他音频」 |

### 8.2 错误码与降级策略

| 错误码 | 场景 | 处理 |
|---|---|---|
| FFMPEG_LOAD_FAIL | ffmpeg.wasm 加载失败 | 提示刷新页面或切换浏览器（需 WebAssembly 支持） |
| FFMPEG_EXTRACT_FAIL | 音频提取失败（文件损坏） | 提示文件可能损坏，建议重新上传 |
| BASIC_PITCH_LOAD_FAIL | Basic Pitch 模型加载失败 | 提示网络问题，模型 CDN 加载失败，可重试 |
| BASIC_PITCH_EMPTY | 未提取到任何音符 | 提示音频可能无人声/乐器旋律，建议换曲 |
| SOUNDFONT_LOAD_FAIL | SoundFont 加载失败 | 播放功能不可用，但谱面生成和导出仍正常工作 |
| LOCALSTORAGE_FULL | 曲谱库存储满 | 提示清理旧曲谱或导出 JSON 备份 |
| BROWSER_NOT_SUPPORTED | 浏览器不支持 WebAssembly/Web Audio | 提示使用 Chrome/Edge/Firefox 最新版 |
